What is LangChain?

LangChain is a Python library (pip install langchain) — technically a library, but called a framework because it gives you a structured way to build LLM applications by chaining components: prompts → LLM → memory → tools → output parsers.

Why it exists: Before LangChain, connecting an LLM to a vector DB, memory, and tools required writing all the glue code yourself. LangChain standardised those patterns so you don't reinvent the wheel.

One line summary: Ready-made building blocks for any LLM application — retrieval, memory, agents, tools, and more.

Who does NOT use LangChain?
✗ChatGPT

Built entirely in-house by OpenAI. LangChain is for devs building ON TOP of it.

✗Claude / Claude Code

Anthropic uses their own SDK directly. No LangChain involved.

✗Simple one-off tasks

One prompt, one response, done. LangChain adds unnecessary overhead.

Simple Rule

LangChain is for consumers of LLM APIs, not producers. Use it when building Gen AI apps or AI Agents on top of existing LLMs.
For complex multi-agent orchestration, add LangGraph — it's part of the same ecosystem, not a replacement.
