RAG Strategies — At a Glance

A quick-reference comparison of all five retrieval strategies implemented in this app — from the simplest baseline to the most sophisticated.

Strategy Comparison
Strategy Query Handling Retrieval Method Reranking Best For Avoid When
1. Naive RAG Raw query, no processing Dense vector (cosine) None Prototypes, simple docs Complex/multi-part questions

Strategy Summaries
1 Naive RAG

Split documents into chunks → convert to vectors → store in vector DB → user asks a question → find closest matching chunks → send to LLM → get answer.

That's it. No intelligence in between.

all-MiniLM-L6-v2 cosine similarity fixed chunking
2 Hybrid RAG

Two searches run in parallel:

Dense search — finds chunks by meaning (semantic/embeddings)
Sparse search — finds chunks by exact keywords (BM25)

Both results are merged using RRF.

dense + sparse RRF fusion TF-IDF BM25
3 Advanced RAG

Same as Naive, but smarter at every step:

Before retrieval → clean up / rewrite the user's question first
During retrieval → fetch better, more relevant chunks via hybrid search
After retrieval → remove noise, re-rank results before sending to LLM

query rewrite hybrid search LLM reranking RRF
4 Graph RAG

Documents are stored as a sequential knowledge graph. Retrieval starts with vector-matched seeds, then expands via BFS to pull in neighbouring chunks — ideal when answers depend on surrounding context.

graph nodes BFS expansion NetworkX
5 Agentic RAG

The LLM is not just answering — it's deciding. It asks "do I need to search?", searches, reads the result, asks "is this enough?", searches again if needed, and only then answers.

Multi-step reasoning loop.

Beyond Text Multimodal RAG

In a multimodal RAG system, not only text but also images, audio, and video can be converted into embeddings and stored as vectors in a vector database. Text uses text-embedding models, images use vision-embedding models, audio uses audio-embedding models, and video uses video-embedding models.

All modalities become numerical vectors that can be stored in the same or separate indexes. During retrieval, the user query is embedded and matched against these vectors — returning relevant text, images, audio, or video.

Key Takeaway

"Vector DB is the backbone of most RAG systems, but Agentic RAG is tool-agnostic and Graph RAG augments it with structural graph traversal for context-aware retrieval."
